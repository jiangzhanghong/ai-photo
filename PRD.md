# 光影AI 产品需求文档

## 1. 文档信息

- 产品名称：光影AI
- 产品类型：AI 摄影服务平台
- 当前阶段：静态官网已完成，后续规划会员订阅、积分、AI 图片生成/修改、小程序、后台管理与后端服务
- 目标版本：V1.0
- 文档状态：草案
- 输出目录：项目根目录 `/app/ai-photo`

## 2. 产品背景

光影AI 面向个人写真、商品摄影、品牌视觉和商业定制用户，提供基于摄影师服务与 AI 后期能力结合的影像解决方案。当前仓库已有静态官网页面，用于展示品牌定位、精选作品、风格专题、价格方案、创作者、客户评价和预约咨询入口。

后续产品需要从单一展示页扩展为可获客、可订阅、可生成/修改图片、可下单、可管理、可交付的业务系统，支撑从用户咨询、会员订阅、积分获取、AI 创作、套餐选择、资料提交、订单跟进、作品交付到客户评价的完整服务流程。

## 3. 产品目标

1. 建立清晰可信的 AI 摄影品牌入口，提高用户咨询转化。
2. 支持用户浏览作品、选择风格、查看套餐并提交预约。
3. 支持会员订阅获取积分，并通过网站消耗积分生成或修改图片。
4. 支持用户通过验证码完成注册和登录，首期实现邮箱验证码注册与登录，后续支持手机号验证码。
5. 支持运营人员管理作品、套餐、会员方案、积分、核心提示词、生图模型、创作者、预约线索和订单状态。
6. 支持后续接入 AI 修图、风格化生成、图像交付和客户反馈能力。
7. 为小程序、管理后台和后端 API 预留一致的数据模型和业务流程。

## 4. 用户角色

### 4.1 访客用户

未登录用户，可以浏览官网或小程序内容，查看作品、价格、创作者与客户评价，并提交咨询预约。

核心诉求：

- 快速理解服务是否符合需求。
- 查看作品质量和风格。
- 明确价格、交付周期和服务内容。
- 低门槛提交咨询。

### 4.2 注册客户

已留资或登录用户，可以通过邮箱验证码注册和登录，注册完成后设置密码，后续可通过密码登录；忘记密码时通过验证码重设密码并重新登录。手机号验证码注册和登录作为后续能力预留。登录后用户可以订阅会员、获取积分、使用提示词标题生成或修改图片、创建订单、上传原始图片、查看服务进度、接收成片和提交评价。

核心诉求：

- 选择合适套餐和风格。
- 通过会员订阅获得稳定的图片生成/修改额度。
- 使用平台预设提示词完成创作，但不接触核心提示词内容。
- 安全上传照片和需求说明。
- 清楚了解订单进度。
- 下载最终成片。

### 4.3 创作者 / 摄影师

平台合作摄影师或 AI 视觉创作者，可以维护个人资料、作品案例和接单信息。

核心诉求：

- 展示专业能力。
- 获取预约线索。
- 管理项目进度。
- 上传交付作品。

### 4.4 运营管理员

平台运营人员，负责内容、线索、订单、套餐、会员、积分、提示词、创作者和评价管理。

核心诉求：

- 快速更新官网展示内容。
- 跟进预约咨询。
- 管理订单状态和交付。
- 管理会员订阅、积分发放和消耗记录。
- 维护核心提示词，控制前台可见标题和后台实际 prompt 内容。
- 维护价格套餐和风格分类。

## 5. 使用场景

### 5.1 个人写真预约

用户浏览人像作品和风格专题，选择免费方案或标准版套餐，提交姓名、手机号、拍摄类型、期望风格和预约时间。运营人员联系确认，转化为订单。

### 5.2 商品图 AI 精修

商家查看商品或静物作品，选择标准版或高级版，上传商品原图和参考风格，平台完成 AI 调色、背景优化和精修后交付。

### 5.3 品牌商业定制

企业客户通过定制版入口提交品牌需求，运营人员创建项目，安排摄影师和 AI 视觉团队，按项目节点交付方案、样片和最终成片。

### 5.4 内容运营更新

运营管理员在后台上传新作品、维护首页推荐、调整价格方案、上下架创作者和精选评价，前台页面自动展示最新内容。

### 5.5 会员积分 AI 生图

用户订阅会员后获得积分，在网站选择图片生成类型、提示词标题、尺寸和数量，提交生成任务。系统扣减积分并调用 AI 能力生成图片，用户可在创作记录中查看、下载和再次编辑。

### 5.6 会员积分 AI 改图

用户上传原图，选择修改类型和提示词标题，补充可选说明后提交任务。系统根据后台配置的核心提示词和用户输入生成 AI 处理任务，扣减积分并返回修改后的图片。

### 5.7 核心提示词运营

管理员在后台维护核心提示词，包括标题、适用场景、任务类型、实际 prompt 内容、消耗积分、排序和上下架状态。前台用户只能看到提示词标题和必要说明，不能查看、复制或通过接口获取实际 prompt 内容。

### 5.8 验证码注册与登录

用户通过邮箱输入验证码完成注册和验证码登录。注册时用户填写邮箱并获取验证码，系统校验验证码和密码复杂度后创建用户账号并完成登录；登录时用户填写已注册邮箱并获取验证码，系统校验后登录。注册完成后用户必须设置密码，后续支持邮箱加密码登录。忘记密码时，用户通过邮箱验证码重设密码，重设成功后需要重新登录。手机号验证码注册和登录需要在产品和数据模型层面预留字段与接口，但首期暂不接入短信服务。

### 5.9 生图模型后台维护

管理员在后台维护可用于图片生成和图片修改的 AI 模型，包括模型名称、供应商、模型编码、支持任务类型、默认尺寸、默认参数、积分消耗配置、成本配置、上下架状态和版本说明。前台用户可以选择已上架模型，默认模型为豆包；用户选择模型后，系统记住该用户最近一次选择，并在后续进入 AI 创作页时默认选中。

## 6. 功能范围

### 6.1 官网 Web

当前已有静态页面，V1.0 需要保留并完善以下模块：

- 首页品牌展示：品牌名、定位语、主视觉图、预约入口。
- 作品集：支持作品分类展示，包括全部、人像、风光、静物、纪实、婚礼、旅行。
- 风格专题：支持极简、电影感、法式、韩系、复古等风格展示。
- 价格方案：展示免费方案、标准版、高级版、定制版套餐，套餐价格、赠送积分、可生成图片张数和版本信息均由后端维护。
- 会员订阅：展示会员方案、可获得积分、有效期和权益说明。
- 注册登录：首期支持邮箱验证码注册和验证码登录；注册完成后设置密码；后续支持邮箱密码登录和忘记密码重设。手机号验证码入口预留但不作为首期交付。
- AI 创作入口：支持登录用户选择提示词标题，进行图片生成或图片修改。
- 创作记录：支持用户查看 AI 任务状态、积分消耗、结果图片和下载入口。
- 创作者推荐：展示摄影师或视觉创作者头像、名称、擅长方向和作品入口。
- 客户评价：展示客户头像、身份和评价内容。
- 预约咨询：提供咨询入口，后续接入表单或小程序二维码。
- 页脚信息：展示联系方式、服务导航、帮助中心、公众号二维码和备案信息。

V1.0 增强需求：

- 顶部导航应滚动到对应模块。
- 所有图片资源加载失败时应有合理替代文本。
- 预约按钮应进入可提交表单或可追踪的联系方式。
- AI 创作入口应要求用户登录，并在积分不足时引导订阅或购买积分。
- 注册和验证码登录均应使用验证码校验，首期验证码通过邮箱发送；注册完成后必须设置密码，后续支持密码登录。
- 登录态采用 JWT + refresh token + Redis，Redis 地址和密码从环境变量 `redis_addr`、`redis_password` 读取。
- 前台只展示提示词标题、分类和适用场景，不展示核心提示词正文。
- 页面应适配移动端，当前固定宽度页面后续需要响应式改造。
- 页面内容应支持后台配置，而不是硬编码在 HTML 中。

### 6.2 小程序端

小程序用于承接移动端获客、预约和订单跟进。

核心功能：

- 首页：品牌介绍、推荐作品、热门风格、套餐入口。
- 作品列表：按分类筛选作品，支持作品详情。
- 风格选择：展示风格说明、示例图和适用场景。
- 套餐选择：展示价格、赠送积分、可生成图片张数、精修张数、交付时间和服务内容。
- 会员订阅：展示会员方案、积分额度、有效期和购买入口。
- AI 创作：支持选择提示词标题生成图片，或上传图片后选择提示词标题进行修改。
- 预约表单：填写姓名、手机号、微信号、需求类型、预算、期望时间、备注。
- 图片上传：支持上传原片、参考图和需求说明。
- 创作记录：查看 AI 生成/修改任务状态、积分消耗、结果图和失败原因。
- 订单列表：查看订单状态、套餐、价格和交付时间。
- 订单详情：查看服务进度、沟通记录、成片下载和评价入口。
- 用户中心：个人资料、联系方式、历史订单、隐私授权。

### 6.3 后台管理

后台用于运营管理和业务处理。

核心功能：

- 登录与权限：管理员登录，区分超级管理员、运营、创作者等角色。
- 用户账号管理：查看用户邮箱、手机号、注册来源、密码设置状态、登录时间、状态和会员信息。
- 作品管理：新增、编辑、删除、排序、分类、推荐作品。
- 风格管理：维护风格名称、英文名、封面图、说明和展示状态。
- 套餐管理：维护套餐名、版本号、价格、赠送积分、可生成图片张数、精修张数、交付周期、权益和上下架状态。
- 会员方案管理：维护会员名称、价格、赠送积分、有效期、权益和上下架状态。
- 积分管理：查看用户积分余额、订阅发放记录、任务消耗记录和人工调整记录。
- 核心提示词管理：维护提示词标题、任务类型、适用场景、实际 prompt 内容、消耗积分、排序、版本和上下架状态。
- 生图模型管理：维护 AI 模型供应商、模型编码、支持任务类型、默认参数、积分消耗、成本、版本、上下架状态和默认选择规则；默认模型为豆包。
- AI 任务管理：查看图片生成/修改任务、用户、提示词标题、状态、消耗积分、输入图、结果图和失败原因。
- 创作者管理：维护创作者头像、姓名、头衔、擅长方向、作品关联和展示状态。
- 预约线索管理：查看用户预约信息，标记跟进状态，转化为订单。
- 订单管理：创建订单、分配创作者、修改状态、上传交付文件、记录备注。
- 评价管理：审核客户评价，选择是否展示到前台。
- 素材管理：统一管理图片、二维码、交付文件和下载链接。
- 系统设置：维护品牌信息、联系方式、备案号、公众号二维码等。

### 6.4 后端服务

后端提供统一 API、数据存储和业务状态流转。

核心能力：

- 用户与管理员认证。
- 用户邮箱验证码发送、校验、注册、验证码登录、密码登录和忘记密码重设 API，预留手机号验证码能力。
- 内容管理 API。
- 套餐方案 API，支持前台获取后端维护的套餐版本、价格、赠送积分和可生成图片张数。
- 预约线索 API。
- 订单和交付 API。
- 会员订阅和积分账户 API。
- AI 图片生成和修改任务 API。
- 核心提示词 API，前台接口不得返回实际 prompt 内容。
- 生图模型管理 API，仅后台可维护；前台不得获取模型敏感参数。
- 图片上传和文件存储。
- 数据权限控制。
- 操作日志。
- 后续 AI 处理任务队列。

## 7. 业务流程

### 7.0 注册与登录流程

1. 用户进入注册或登录页面。
2. 用户填写邮箱并点击获取验证码。
3. 系统校验邮箱格式、发送频率和当日发送次数。
4. 系统生成验证码，写入验证码记录，并通过邮件发送。
5. 注册场景下，用户提交邮箱、验证码和密码。
6. 系统校验验证码是否正确、未过期、未使用，并校验密码复杂度。
7. 邮箱未注册时，系统创建用户、写入密码哈希、标记邮箱已验证并完成登录；邮箱已注册时提示去登录。
8. 验证码登录场景下，用户提交邮箱和验证码；系统校验通过后签发 JWT access token 和 refresh token。
9. 密码登录场景下，用户提交邮箱和密码；系统校验密码正确后签发 JWT access token 和 refresh token。
10. 忘记密码场景下，用户通过邮箱验证码重设密码；重设成功后清理旧 refresh token，要求用户重新登录。
11. 登录态使用 Redis 保存 refresh token 状态、失效时间和用户会话信息，Redis 连接信息来自环境变量 `redis_addr`、`redis_password`。
12. 验证码校验成功后标记为已使用，并记录用户登录时间。

### 7.1 预约咨询流程

1. 用户浏览官网或小程序。
2. 用户选择服务类型或套餐。
3. 用户提交预约表单。
4. 系统生成预约线索，状态为“待跟进”。
5. 运营人员联系用户，补充需求。
6. 线索转化为订单或标记为无效。

### 7.2 订单服务流程

1. 运营创建订单或用户自助下单。
2. 用户上传原片和参考图。
3. 运营确认需求和交付时间。
4. 分配摄影师或 AI 视觉创作者。
5. 创作者上传样片或初稿。
6. 用户确认或提出修改意见。
7. 创作者上传最终成片。
8. 订单完成，用户提交评价。

### 7.3 内容发布流程

1. 运营在后台创建或编辑内容。
2. 内容保存为草稿。
3. 运营预览前台展示效果。
4. 内容发布上线。
5. 前台官网和小程序同步展示。

### 7.4 会员订阅与积分流程

1. 用户登录网站或小程序。
2. 用户选择会员方案。
3. 用户完成支付或由管理员人工开通。
4. 系统为用户开通会员权益并发放积分。
5. 用户在 AI 创作、AI 改图或其他服务中消耗积分。
6. 系统记录每一笔积分发放、消耗、退款或人工调整。

### 7.5 AI 图片生成流程

1. 用户进入 AI 创作页面。
2. 用户选择任务类型、提示词标题、生图模型、图片尺寸和生成数量；模型默认选中用户最近一次选择，没有历史选择时默认豆包。
3. 系统校验登录状态、会员状态、模型状态和积分余额。
4. 系统读取后台配置的核心提示词正文，结合用户可选输入生成任务请求。
5. 系统按所选模型和任务参数计算积分消耗，扣减积分并创建 AI 任务，状态为“排队中”。
6. AI 服务返回结果后，系统保存图片并更新任务状态。
7. 用户查看、下载结果图，也可以基于结果图继续发起修改任务。

### 7.6 AI 图片修改流程

1. 用户上传需要修改的原图。
2. 用户选择修改类型、提示词标题和生图模型，可填写补充说明。
3. 系统校验文件格式、大小、会员状态、模型状态和积分余额。
4. 系统使用后台核心提示词正文和用户补充说明创建改图任务。
5. 系统按所选模型和任务参数计算积分消耗，扣减积分并提交 AI 服务处理。
6. AI 服务返回修改结果后，用户查看、下载或再次修改。

### 7.7 核心提示词使用流程

1. 管理员在后台创建或编辑核心提示词。
2. 管理员填写前台可见标题、任务类型、适用场景和后台实际 prompt 内容。
3. 管理员设置消耗积分、排序和上下架状态。
4. 前台只展示已上架提示词的标题、分类和适用说明。
5. 用户提交任务时只传递提示词 ID，后端根据 ID 读取实际 prompt 内容。
6. 前台页面、浏览器响应和公开 API 均不得返回实际 prompt 内容。

### 7.8 生图模型选择流程

1. 管理员在后台创建或编辑 AI 模型配置。
2. 管理员设置模型支持的任务类型、默认参数、积分消耗、成本、版本和上下架状态。
3. 管理员将豆包设置为默认模型，也可以调整不同任务类型的默认模型。
4. 用户提交 AI 生图或改图任务时，前台传任务类型、提示词 ID、模型 ID、尺寸、数量、输入图和补充说明。
5. 后端校验模型是否上架且支持当前任务类型，并按模型积分配置计算本次任务消耗。
6. 用户提交成功后，系统记住该用户最近一次选择的模型。
7. AI 任务记录保存所使用的模型 ID 和模型版本，便于成本统计、问题排查和结果追踪。

## 10. 非功能需求

### 10.1 性能

- 官网首屏静态资源应尽量压缩，图片使用合适尺寸。
- 首页核心内容在普通 4G 网络下应尽快可见。
- 后台列表查询需要分页。
- AI 任务提交后应异步处理，避免页面长时间阻塞。

### 10.2 安全

- 后台必须登录后访问。
- 管理接口需要权限校验。
- 邮箱验证码和手机号验证码不得明文存储，只保存哈希值。
- 验证码发送、校验和登录接口需要频率限制、错误次数限制和 IP 风控。
- 用户密码不得明文存储，需要使用安全哈希算法保存。
- 密码复杂度默认要求为 8-32 位，至少包含大小写字母、数字、特殊字符中的 3 类，且不能包含邮箱前缀。
- 登录态使用 JWT access token + refresh token + Redis。access token 短有效期，refresh token 状态存 Redis，退出登录、忘记密码重设和风控封禁时应使 refresh token 失效。
- Redis 连接信息必须通过环境变量 `redis_addr`、`redis_password` 配置。
- 上传文件需要限制类型和大小。
- 用户隐私信息如手机号、微信号应避免在日志中明文输出。
- 核心提示词正文、反向提示词和模型参数属于后台敏感配置，不得返回给前台。
- AI 任务日志不得记录完整核心提示词内容。
- 积分扣减和退款需要保证幂等，避免重复扣费或重复退款。
- 数据库账号应使用最小权限账号，避免远程 root。

### 10.3 可用性

- 表单字段需要明确错误提示。
- 订单状态需要清晰展示。
- 图片上传需要显示进度和失败重试。
- AI 任务失败时需要展示明确原因，并按规则退回积分。
- 提示词标题需要让用户理解效果方向，但不能暴露具体 prompt 内容。
- 管理后台的删除操作需要二次确认。

### 10.4 兼容性

- 官网支持主流桌面浏览器和移动浏览器。
- 小程序优先支持微信小程序。
- 后台优先支持 Chrome 最新稳定版。

## 14. 风险与待确认事项

### 14.1 风险

- 当前前端页面为固定宽度，移动端体验不足。
- 当前内容硬编码，后续动态化需要重构数据来源。
- 图片和成片涉及用户隐私，需要明确存储、访问和删除策略。
- AI 处理流程尚未定义，后续需明确模型、成本、耗时和失败补偿机制。
- 邮件验证码依赖邮件服务商，需关注到达率、延迟和垃圾邮件拦截。
- 会员订阅和积分涉及支付、退款和对账，需要明确支付渠道和财务规则。
- 核心提示词属于平台资产，需要避免前台泄露和日志泄露。
- AI 生成内容需要审核和风控，避免生成违规图片。
- 远程数据库账号需要最小权限配置，避免使用 root。

### 14.2 待确认

- 是否需要在线支付，以及首期接入微信支付、支付宝还是人工开通。
- 积分是否支持单独购买，还是仅通过套餐和会员订阅获得。
- AI 任务失败后是否自动退回积分，或需要人工审核。
- 不同提示词是否允许设置不同积分消耗。
- 是否需要对用户输入和生成结果做内容安全审核。
- 是否需要用户登录后才能上传图片。
- 成片交付是否支持长期保存。
- 是否需要多创作者接单和佣金结算。
- 是否需要接入短信通知、微信订阅消息或企业微信通知。
- AI 修图是平台自研、第三方 API，还是人工与 AI 混合流程。
- 邮箱验证码服务商、发件域名和邮件模板是否已有指定。
- 手机号验证码后续是否接入短信服务商，以及是否和邮箱验证码共用注册登录入口。
- 用户注册时邮箱已存在，是否允许验证码校验后直接登录。
- 密码复杂度是否采用默认规则：8-32 位，至少包含大小写字母、数字、特殊字符中的 3 类，且不能包含邮箱前缀。
- 生图模型默认使用豆包，是否还需要按不同会员等级限制可选模型。
